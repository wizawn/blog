## 🔍 Hexo 7.x 程序化构建关键发现 (2:35 PM UTC)

### 核心问题与解决方案

**问题 1: 插件不加载**
- **现象**: `load_plugins.js` 守卫 `if (!ctx.env.init || ctx.env.safe) return;` 阻止插件加载
- **解决**: 在 `hexo.init()` 之前设置 `hexo.env.init = true`

**问题 2: hexo-renderer-marked 不注册**
- **现象**: Markdown 渲染器未加载
- **解决**: 设置 `global.hexo = hexo` 在加载插件前

**问题 3: 主题视图为空 (hexo.theme.views = 0)**
- **现象**: `Template render error: template not found: _layout.njk`
- **根因**: Hexo 7.x 的 `hexo.init()` **不会自动调用** `hexo.theme.process()`
- **解决**: 手动调用 `await hexo.theme.process()` 在 generate 之前

### 正确的构建脚本顺序

```javascript
const hexo = new Hexo(process.cwd(), {});
global.hexo = hexo;
hexo.env.init = true;
await hexo.init();
await hexo.source.process();      // 处理源文件
await hexo.theme.process();       // ⚠️ 必须手动调用！注册主题视图
await hexo.call('generate', {});  // 生成静态文件
```

### 其他发现
- **生成器调用**: 必须用 `hexo.call('generate')` 保持正确的 `this` 上下文
- **NexT 主题**: 需要 `hexo-renderer-nunjucks` 渲染 `.njk` 模板
- **文件名陷阱**: Git 可能产生 NBSP 字符 (`\302\240`) 在文件名中，用 `git checkout HEAD -- source/` 修复
- **验证**: 修复后 views 从 0 → 72，`_layout.njk` 可正确获取

### Cloudflare 部署阻塞
- **Token 权限不足**: 缺少 `/accounts`, `/pages/projects` 端点权限
- **错误码**: 10000, 7000, 9109
- **需要**: 重新生成带 Pages 部署权限的 API Token
