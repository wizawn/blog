# 博客去重检查清单

## 🚨 重复内容识别

### 常见重复场景
1. **RSS 源重复** - 同一内容在不同时间抓取
2. **标题高度相似** - 仅标点/时间戳差异
3. **URL 相同** - 最准确的重复标识
4. **内容雷同** - 摘要相似度 >85%

### 重复案例
| 文章标题 | 重复次数 | 根因 |
|----------|----------|------|
| 套壳中国大模型撑起 500 亿美元估值 | 13 次 | RSS 源循环 |
| 零安装的云养虾：ArkClaw 使用指南 | 10 次 | RSS 源循环 |
| 科技爱好者周刊（第 389 期） | 14 次 | RSS 源循环 |

## ✅ 发布前检查（必须执行）

### 1. URL 去重（最准确）
```bash
# 检查现有文章的 URL
grep -r "link:" source/posts/auto-*.md | sort -u
```

**标准**：相同 URL 直接跳过

### 2. 标题精确匹配
```bash
# 检查现有文章标题
grep "^title:" source/posts/auto-*.md | sort | uniq -c | sort -rn
```

**标准**：完全相同标题跳过

### 3. 标题模糊匹配
```bash
# 运行去重脚本
python3 /tmp/dedup_blog.py
```

**标准**：前 50 字符相似度 >90% 跳过

### 4. 内容相似度
```bash
# 检查内容摘要
grep -A5 "## 摘要" source/posts/auto-*.md
```

**标准**：摘要相似度 >85% 跳过

## 📝 发布流程

### 正常发布
1. RSS 抓取 → 2. 去重检查 → 3. 生成文章 → 4. 部署

### 发现重复
1. 跳过重复内容
2. 日志记录原因
3. 继续处理其他文章
4. 发布后检查重复率

### 发布后检查
```bash

ls -lt source/posts/auto-*.md | head -5

# 检查是否有重复
python3 /tmp/dedup_blog.py --dry-run
```

## ⚠️ 经验教训

### 2026-03-26 去重行动
- **删除**：105 篇重复文章
- **保留**：182 篇唯一文章

### 关键发现
1. RSS 源内容有限 → 需要更多源
2. 标题不可靠 → URL 最准确
3. 时间戳陷阱 → 相同内容不同时间
4. 预防胜于治疗 → 发布前去重 > 事后清理

### 预防措施
1. ✅ URL 去重（最准确）

3. ✅ 内容相似度检测
