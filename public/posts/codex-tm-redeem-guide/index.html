CodeX TM 兑换码使用教程 - 详细步骤与常见问题解答

Wednesday, March 25, 2026

📋 前言

本文详细介绍 CodeX TM 兑换码的购买、兑换及质保流程，帮助用户顺利完成兑换并加入工作空间。包含完整的图文教程、席位额度说明和常见问题解答。

⚠️ 兑换前注意事项

🚨 重要时段提醒（新增）

⚠️ 避免北京时间 14:00 左右兑换

根据大量用户反馈和观察数据，北京时间 14:00 前后（对应美国午夜时段）封号情况极为严重。

建议：

  • ❌ 避免此时段兑换新号
  • ❌ 避免此时段补车/质保换号
  • ✅ 建议选择白天时段（北京时间 9:00-12:00 或 15:00-18:00）
  • ✅ 如必须此时段操作，请做好心理准备和质保预案

原因分析：官方在午夜时段进行大规模风控审查，新兑换账号容易被批量封禁。

1. 购买多个兑换码

  • 重要：购买多个号的请注意间隔一段时间再兑换
  • 原因：连续兑换可能导致多个账号加入同一个空间
  • 建议：每个兑换码间隔 5-10 分钟

2. 兑换失败处理

  • 如遇兑换失败，通常是库存不足
  • 系统正在补货中
  • 请等待10 分钟左右再重试

3. 邮箱填写

  • ⚠️ 请务必填写正确的邮箱地址
  • 如填错无法补回邀请邮件
  • 兑换后请在邮箱内根据引导加入空间

🔗 兑换链接

https://codextm.xyz

注意：将 - 替换为 . 访问

📝 操作步骤

第一步：访问兑换链接

打开浏览器访问 https://codextm.xyz

第二步：输入邮箱地址

  • 填写您用于注册的正确邮箱
  • ⚠️ 请仔细核对邮箱地址，确保无误

第三步：输入兑换码

  • 将购买获得的兑换码复制到输入框
  • 确保没有多余空格或字符

第四步：验证并兑换

  • 完成验证码验证
  • 点击"兑换"按钮
  • 等待系统处理

第五步：检查邮箱

  • 兑换成功后，系统会发送邀请邮件到您的邮箱
  • 根据邮件内的引导加入工作空间
  • ⚠️ 如未收到邮件，请检查垃圾邮件箱

🛡️ 质保步骤

质保流程与兑换流程相同：

  1. 访问兑换链接 https://codextm.xyz
  2. 选择 “质保兑换” 选项
  3. 填写邮箱地址
  4. 输入质保兑换码
  5. 验证并完成质保

  • 每个席位均为独立额度，互不共享、互不影响
  • 单独使用不受其他席位干扰

2. 官方固定额度

  • 每周 5 小时等效使用时长
  • 为平台硬性规则，无法调整或提升

3. 滚动刷新机制

  • 采用 5 小时滚动刷新机制
  • 每 5 小时恢复对应使用额度
  • 非固定自然周重置

4. 并发任务

  • 支持最多 2 个任务同时运行
  • 并发使用更高效

备份建议

方法二：手动复制保存

  • 定期复制重要对话内容
  • 保存到本地文档或云笔记
  • 建议按日期或项目分类存储

在任务开始时明确要求：

🔧 常见问题解决方案

一、如何兑换激活码

兑换步骤：

  1. 进入首页，选择【兑换激活码】
  2. 输入兑换码和邮箱
  3. 兑换成功后请在邮箱确认

兑换并确认成功后即可使用服务。

二、质保换号说明

如果空间遭封禁按照以下流程进行换空间：

质保换号流程

  1. 进入"质保换号"页面
  2. 输入原激活码
  3. 输入原邮箱
  4. 点击申请换号

申请兑换成功后请退出原空间，如果使用 Codex 需重新登录清除缓存。

三、邮件收不到怎么办

邮件收不到按照以下步骤：

  1. 在邮件界面刷新等一会看一下
  2. 如果还是没有的话查看一下网站是否兑换/质保成功
  3. 还是不行的话在网站质保换号，换号成功基本都会收到的
  4. 如果次数用完可以联系客服进行更换兑换码
  5. 如还是不能解决请联系客服

四、401 Unauthorized（未授权）

如果出现 401 一般有两个问题：

  1. 配置文件问题：如果之前使用过 API 中转的，需要删除配置文件 config.toml 或者重新配置

    • 可以使用 cc-switch 直接更换官方配置（需要重新登录）
    • CC-SWITCH 配置请查看相关教程

  2. 邮箱确认问题：没有通过邮箱确认，在收到邀请了请确保进行确认，如果收不到邮箱重新质保

  3. 空间选择问题：请确保登录选择的空间能正常使用，最好手动把停用的空间退出

五、402 Payment Required（支付/配额）

402 就是空间被停用了，这是在使用期间可能遇到的问题，按照以下方法解决：

  1. 到兑换网站质保换号，换号成功并且在邮件确认
  2. 如果是使用 Codex，需要重新登录，在登录的时候请确保选择正常的空间

PS: 防止登录选错，可以手动推掉已被停用的空间

八、Reconnecting 重连超过五次怎么办

问题描述：Codex 频繁重连，显示 “Reconnecting…” 超过 5 次后断开连接。

问题原因：WebSocket 连接不稳定，Codex 默认优先使用 WebSocket 传输，但某些网络环境下 WebSocket 容易断开。

参考 Issue：WebSocket upgrade succeeds then server closes with 1008 Policy (falls back to HTTPS) · Issue #13041 · openai/codex

解决方案：禁用 WebSocket，强制使用 HTTPS

步骤一：修改 config.toml 配置文件

找到 Codex 配置文件（通常位于 ~/.codex/config.toml），添加以下配置：

# 禁用 WebSocket，强制使用 HTTP
model_provider = "openai_http"

[model_providers.openai_http]
name = "OpenAI HTTP only"
wire_api = "responses"
requires_openai_auth = true
supports_websockets = false
base_url = "https://chatgpt.com/backend-api/codex"

步骤二：修改现有对话记录的 provider

由于 provider 变更，对话记录会清空，但数据仍保存在本地。需要让 Codex 自动迁移对话记录：

# 找到对话记录 JSONL 文件
cd ~/.codex/sessions/

# 备份现有对话记录（可选但推荐）
cp -r sessions/ sessions_backup/

# 让 Codex 自动迁移对话记录
# 方法：修改对话记录中的 provider 字段
find . -name "*.jsonl" -exec sed -i 's/"provider": "openai"/"provider": "openai_http"/g' {} \;

步骤三：重启 Codex

# 完全退出 Codex
codex logout

# 重新启动 Codex
codex

步骤四：验证配置

启动后执行以下命令验证配置是否生效：

# 检查当前配置
codex config show

# 验证连接状态
codex whoami

注意事项

  1. 对话记录迁移：provider 变更后，Codex 会自动查找并迁移本地对话记录，但建议先备份
  2. 配置持久化：修改后的 config.toml 会持久化保存，无需每次重启都修改
  3. 网络兼容性：此方案适用于 WebSocket 被防火墙或不稳定网络环境
  4. 性能影响：HTTP 传输相比 WebSocket 可能有轻微延迟，但稳定性更好

如果问题仍未解决

  1. 检查代理配置是否正确
  2. 尝试切换代理节点
  3. 清理 .codex/ 目录后重新登录
  4. 联系技术支持反馈具体错误日志

通用排查步骤（必做）

  1. 退出所有账号，清理缓存，重启终端/编辑器
  2. 用无痕模式登录，确保代理全局生效
  3. 登录后执行 codex whoami 验证身份
  4. 若仍报错，删除 .codex/ 目录后重试

📧 邮件收不到怎么办？

可能原因

  1. 邮箱地址填写错误
  2. 邮件被归类到垃圾邮件
  3. 邮件服务器延迟
  4. 邮箱已满

解决方案

  1. 检查垃圾邮件箱：查看垃圾邮件/广告邮件文件夹
  2. 核对邮箱地址：确认兑换时填写的邮箱地址正确
  3. 等待一段时间：邮件可能有延迟，等待 10-30 分钟
  4. 清理邮箱空间：确保邮箱有足够的存储空间
  5. 联系管理员：如以上方法都无效，请留言提供邮箱地址
