# ARL 灯塔实战：10,000+ 海外站点资产收集全记录

> ⚠️ **免责声明**：本文仅用于安全研究与教育目的，请勿将技术用于非法用途。所有扫描目标均为公开可访问的海外站点，符合 SRC 挖洞规范。

## 📋 项目背景

海外站点（尤其是中小企业、SaaS 服务、电商平台）一直是 SRC 挖洞的优质目标。相比国内大厂，海外站点普遍存在：
- 安全意识相对薄弱
- 技术栈多样化
- 漏洞响应速度快
- 赏金支付及时

本次实战使用 **ARL（Asset Reconnaissance Lighthouse）资产侦察灯塔系统**，对海外站点进行系统性资产收集，为后续 Nuclei 漏洞扫描打下基础。

## 🎯 目标设定

1. **覆盖范围**：北美、欧洲、澳洲等地区企业站点
2. **收集目标**：域名、子域名、IP、站点、证书
3. **预期规模**：10,000+ 域名，10,000+ 站点
4. **时间预算**：12-24 小时
5. **合规要求**：仅扫描公开资产，遵守 robots 协议

## 🛠️ 工具选型

### ARL 灯塔系统

**GitHub**: https://github.com/TophantTechnology/ARL

ARL 是知道创宇开发的资产侦察工具，支持：
- 域名爆破
- 端口扫描
- 站点识别
- 证书收集
- Nuclei 集成
- 任务调度

**部署方式**：Docker Compose

```yaml
version: '3'
services:
  arl_web:
    image: honmashironeko/arl-docker-all
    ports:
      - "5003:443"
    volumes:
      - ./data:/data
```

### Nuclei

**GitHub**: https://github.com/projectdiscovery/nuclei

Nuclei 是基于模板的漏洞扫描器，支持：
- 12,000+ 社区模板
- 自定义 POC
- 批量扫描
- 结果导出

### FOFA / Shodan

用于初始资产发现：
- FOFA: `country="US" && server="nginx"`
- Shodan: `country:US port:443`

## 📊 实战过程

### 阶段一：目标收集（11:30-12:30 UTC）

#### 1. FOFA 查询

使用 FOFA 搜索海外站点：

```
# 美国企业站点
query: country="US" && server="nginx" && title="login"
结果：500,000+ 条

# 电商平台
query: body="shopping cart" && country="US"
结果：200,000+ 条

# SaaS 服务
query: title="dashboard" && server="cloudflare"
结果：300,000+ 条
```

#### 2. 目标筛选

手动筛选高价值目标：

| 类型 | 筛选条件 | 优先级 |
|------|----------|--------|
| 电商平台 | 有登录页面、购物车 | 高 |
| SaaS 服务 | 有 Dashboard、用户系统 | 高 |
| 企业官网 | 有联系表单、新闻系统 | 中 |
| 博客/CMS | WordPress、Joomla 等 | 中 |
| API 服务 | 有 API 文档、Swagger | 高 |

**总计**：50 个扫描任务

### 阶段二：资产收集（12:30-18:00 UTC）

ARL 自动执行以下流程：

1. **域名爆破** - 使用字典爆破子域名
2. **DNS 查询插件** - 调用 crt.sh、Virustotal 等 API
3. **端口扫描** - Top 100 端口
4. **站点识别** - HTTP/HTTPS 站点探测
5. **证书收集** - SSL 证书信息提取

**实时进度**：

| 时间 | 域名 | 站点 | IP | 证书 |
|------|------|------|----|----|
| 12:09 | 1,200 | 380 | 5,500 | 380 |
| 14:00 | 3,000 | 2,300 | 7,800 | 750 |
| 16:00 | 6,500 | 7,200 | 8,800 | 1,280 |
| 18:00 | 9,000 | 9,500 | 9,500 | 1,680 |
| 22:00 | 10,200 | 11,000 | 9,800 | 1,850 |

### 阶段三：Nuclei 扫描（02:00 UTC 启动）

对收集到的 10,200 个域名执行全量 Nuclei 扫描：

```bash
nuclei -l overseas_sites.txt \
  -t nuclei-templates/ \
  -tags cve,vuln,exploit,rce,sqli,xss,ssrf,lfi,rfi,xxe,csrf \
  -severity critical,high,medium,low \
  -rate-limit 30 \
  -concurrency 8 \
  -jsonl -o overseas_scan.jsonl
```

**扫描配置**：
- 目标：10,200 个域名
- POC：6,229 个（标签过滤后）
- 并发：8
- 速率：30 req/s
- 预计耗时：2-4 小时

## 🔍 技术细节

### 1. 域名爆破策略

使用 ARL 内置字典 + 自定义字典：
- 常见子域名：www, mail, vpn, portal, app, api...
- 服务类型：admin, dashboard, login, shop, store...
- 部门/功能：dev, staging, test, prod, internal...

### 2. 证书信息利用

从 SSL 证书中提取：
- 主题备用名称（SAN）
- 颁发者信息
- 有效期
- 指纹
- 组织名称（用于关联资产）

### 3. 站点识别技巧

- HTTP/HTTPS 双协议探测
- 自动跟随重定向
- 状态码过滤（排除 404、503）
- 标题/指纹提取
- CMS 识别（WordPress, Joomla, Drupal 等）

## 🎯 海外 SRC 挖洞建议

### 优先目标

1. **登录页面** - 弱口令/爆破/验证码绕过
2. **文件上传点** - RCE/文件包含
3. **信息泄露** - 配置/备份文件/.git 目录
4. **越权访问** - IDOR/未授权访问
5. **API 接口** - 未鉴权/参数篡改

### 常见漏洞

| 漏洞类型 | 发现概率 | 赏金范围 |
|----------|----------|----------|
| XSS | 高 | $100-500 |
| SQL 注入 | 中 | $500-2000 |
| 文件上传 | 中 | $500-3000 |
| 未授权访问 | 高 | $100-1000 |
| 信息泄露 | 高 | $50-500 |
| SSRF | 低 | $500-5000 |
| RCE | 低 | $1000-10000 |

### SRC 平台推荐

1. **HackerOne** - 最大平台，项目多，支付及时
2. **Bugcrowd** - 项目质量高，赏金可观
3. **Intigriti** - 欧洲平台，响应快
4. **YesWeHack** - 法国平台，项目多样

### 提交技巧

1. **详细描述** - 复现步骤清晰，附带截图/视频
2. **影响说明** - 说明漏洞危害和业务影响
3. **修复建议** - 提供可行的修复方案
4. **英文报告** - 使用专业术语，语法正确
5. **及时跟进** - 回复 triage 问题，配合验证
