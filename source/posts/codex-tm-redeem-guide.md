## ⚠️ 兑换前注意事项

### 1. 购买多个兑换码
- **重要**：购买多个号的请注意**间隔一段时间再兑换**
- 原因：连续兑换可能导致多个账号加入同一个空间
- 建议：每个兑换码间隔 5-10 分钟

### 2. 兑换失败处理
- 如遇兑换失败，通常是**库存不足**
- 系统正在补货中
- 请等待**10 分钟左右**再重试

### 3. 邮箱填写
- ⚠️ **请务必填写正确的邮箱地址**
- 如填错无法补回邀请邮件
- 兑换后请在邮箱内根据引导加入空间

## 🔧 常见问题解决方案

### 一、401 Unauthorized（未授权）

**原因**：token 过期、凭证无效、缓存冲突、环境变量干扰

**解决方案**：

1. **清理本地认证缓存**
   - Windows：删除 `%USERPROFILE%\.codex\auth\` 下所有文件
   - macOS/Linux：`rm -rf ~/.codex/auth/`

2. **重新登录**
   ```bash
   codex logout
   codex login
   ```

3. **检查环境变量**
   - 删除 `OPENAI_API_KEY` / `OPENAI_BASE_URL`
   - 重启终端

4. **确认账号权限**
   - 确认账号有 ChatGPT Plus/Pro 权限（Codex 需订阅）

---

### 二、402 Payment Required（支付/配额）

**原因**：订阅过期、额度耗尽、账号未开通 Codex 权限

**解决方案**：

1. 登录 OpenAI 账号，确认 Plus 订阅有效、无欠费
2. 检查 API/模型额度是否用尽
3. 重新登录并切换到正确的付费账号

---

### 三、不支持的地区（unsupported_country_region_territory）

**原因**：IP 不在支持列表、代理/出口不一致、Node.js fetch 不走代理

**解决方案**：

1. **切换到支持地区 IP**
   - 使用美国/支持地区纯净 IP
   - 避免数据中心/黑名单 IP

2. **确保代理一致**
   - 确保登录与调用用同一出口（浏览器/终端代理一致）

3. **WSL 环境配置**
   ```bash
   sudo apt update && sudo apt install wslu
   wslu <codex login URL>
   ```

4. **强制终端走代理（Linux/macOS）**
   ```bash
   export HTTP_PROXY=http://127.0.0.1:7890
   export HTTPS_PROXY=http://127.0.0.1:7890
   codex login
   ```

---

### 四、不支持的邮箱

**原因**：邮箱未绑定 OpenAI、非 Plus 账号、邮箱类型被风控（如临时邮箱）

**解决方案**：

1. 用绑定了 ChatGPT Plus 的主邮箱登录（不要用子账号/共享号）
2. 避免临时/企业邮箱，优先 Gmail/Outlook 等主流邮箱
3. 登录 OpenAI 官网，确认邮箱已验证、订阅正常
4. 清理浏览器缓存/无痕模式重新登录

---

### 五、CC Switch 配置问题

**问题**：CC Switch 替换中转为官方配置

**解决方案**：
1. 打开 CC Switch 设置
2. 进入配置页面
3. 选择官方配置模板
4. 保存并重启应用

---

### 通用排查步骤（必做）

1. 退出所有账号，清理缓存，重启终端/编辑器
2. 用无痕模式登录，确保代理全局生效
3. 登录后执行 `codex whoami` 验证身份
4. 若仍报错，删除 `.codex/` 目录后重试

## 📧 邮件收不到怎么办？

### 可能原因
1. 邮箱地址填写错误
2. 邮件被归类到垃圾邮件
3. 邮件服务器延迟
4. 邮箱已满

### 解决方案
1. **检查垃圾邮件箱**：查看垃圾邮件/广告邮件文件夹
2. **核对邮箱地址**：确认兑换时填写的邮箱地址正确
3. **等待一段时间**：邮件可能有延迟，等待 10-30 分钟
4. **清理邮箱空间**：确保邮箱有足够的存储空间
5. **联系管理员**：如以上方法都无效，请留言提供邮箱地址
